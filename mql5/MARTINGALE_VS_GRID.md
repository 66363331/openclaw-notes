# 马丁 vs 固定网格 详细对比
## 为什么固定网格能救你的账户

---

## 📊 核心区别

| 对比项 | 马丁 (Martingale) | 固定网格 (Fixed Grid) |
|--------|-------------------|----------------------|
| **加仓手数** | 倍增 (0.01→0.02→0.04→0.08...) | 固定 (0.01→0.01→0.01...) |
| **资金需求** | 指数增长 (需要无限资金) | 线性增长 (可计算) |
| **最大风险** | 理论上无限 (必爆仓) | 有限且可控 |
| **回本速度** | 快 (爆仓前) | 慢但稳 |
| **心理压力** | 极大 (扛单) | 小 (计划内) |
| **适合市场** | 震荡市 (短期) | 所有市场 |

## 💣 马丁的致命问题

### 场景模拟：连续亏损5次

假设：首仓0.01手，每亏30点加仓，止损$220

```
马丁加仓 (2倍增):
第1层: 0.01手 × 30点 = -$3
第2层: 0.02手 × 30点 = -$6  (累计 -$9)
第3层: 0.04手 × 30点 = -$12 (累计 -$21)
第4层: 0.08手 × 30点 = -$24 (累计 -$45)
第5层: 0.16手 × 30点 = -$48 (累计 -$93)
第6层: 0.32手 × 30点 = -$96 (累计 -$189) ⚠️ 接近止损
第7层: 0.64手 × 30点 = -$192 (累计 -$381) 💥 爆仓！

总手数: 1.27手
需要资金: 极大
结果: 爆仓
```

**问题：**
- 只要连续亏损7次，就爆仓
- 黄金XAU/USD一天波动200-300点，7次30点回撤很常见
- 一旦爆仓，之前所有盈利归零

**优势：**

## ⚠️ 马丁的心理陷阱

### 为什么很多人坚持用马丁？

1. **短期看起来有效**
   - 震荡市确实能快速回本
   - 给人"永远不会亏"的错觉

2. **沉没成本谬误**
   - "已经亏了这么多，必须加仓摊平"
   - 越亏越加，直到爆仓

3. **幸存者偏差**
   - 听到有人用马丁赚钱
   - 没听到爆仓的人（他们不说话了）

### 马丁的真实胜率

```
假设每次胜率50%:

连续5次亏损的概率: 3.125%
连续7次亏损的概率: 0.78%
连续10次亏损的概率: 0.098%

看起来概率很低？
但只要你交易足够多次，爆仓是100%必然事件！

数学原理: 大数定律
只要次数足够多，小概率事件必然发生。
```

## ✅ 固定网格的正确使用姿势

### 配置建议 (XAU/USD)

```cpp
input double Inp_FixedLotSize = 0.01;    // 固定0.01手，绝不倍增
input int    Inp_GridPips = 30;          // 每30点加一次
input int    Inp_MaxLayers = 3;          // 最多加3层
input double Inp_MaxTotalLot = 0.05;     // 总手数不超过0.05
input double Inp_HardStopLoss = 200;     // 亏$200全平
```

### 风险控制

1. **总手数硬上限**
   - 即使网格策略也有限制
   - 防止极端行情

2. **硬止损**
   - 亏到$200无条件全平
   - 保住本金最重要

3. **趋势过滤**
   - H1趋势反转时停止加仓
   - 避免逆势死扛

## 🔧 代码关键区别

### 马丁加仓 (危险！)
```cpp
// 马丁：手数倍增
void MartingaleAdd()
{
   double lastLot = GetLastLot();
   double newLot = lastLot * 2;  // 倍增！危险！
   OpenOrder(newLot);
}
```

### 固定网格加仓 (安全)
```cpp
// 固定网格：手数固定
void FixedGridAdd()
{
   double newLot = 0.01;  // 固定，绝不倍增
   OpenOrder(newLot);
}
```

**就这一个区别，决定了爆仓还是存活！**
