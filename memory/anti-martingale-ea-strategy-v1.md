> 配合手动首仓（正向金字塔）的反马丁加仓系统
> 版本：v1.0
> 适用：XAU/USD, EUR/USD 等主流货币对
> 周期：M3/M5/M15（配合你的三层周期）

## 一、系统架构

### 1.1 首仓与EA的关系

```
┌─────────────────────────────────────────────────┐
│              交易系统整体架构                      │
├─────────────────────────────────────────────────┤
│  手动首仓（你操作）                               │
│  ├── 方向判断：H1趋势 + M15结构 + M3进场           │
│  ├── 仓位：正向金字塔（大仓位）                   │
│  ├── 止损：硬止损（如-2%账户）                    │
│  └── 角色：趋势判断 + 主要利润来源                │
│                                                 │
│  EA反马丁加仓（自动）                             │
│  ├── 触发条件：首仓盈利达到加仓间距               │
│  ├── 加仓方向：与首仓同向                         │
│  ├── 仓位：递减（80% → 64% → 51%...）           │
│  ├── 止损：移动保本 + 总账户硬止损                │
│  └── 角色：利润放大器                             │
└─────────────────────────────────────────────────┘
```

### 1.2 核心逻辑

**反马丁核心**：只在盈利时加仓，亏损绝不加仓
**正向金字塔核心**：盈利后减仓锁定利润
**两者配合**：首仓大仓位抓趋势，EA顺势加仓放大趋势利润

### 2.1 首仓入场（手动）

由你根据H1+M15+M3三层周期判断方向并手动开仓

### 2.2 EA识别首仓

```
EA启动时检测：
1. 当前品种是否存在手动开仓的订单
2. 订单开仓时间 < 5分钟（视为新首仓）
3. 记录首仓方向、开仓价、手数
4. 进入监控模式，等待加仓条件
```

### 2.3 加仓触发条件

| 条件 | 说明 | 参数 |
|------|------|------|
| 首仓盈利点数 | 价格向有利方向移动 | AddStartPips = 150 |
| 间距条件 | 每间隔一定点数加仓 | AddSpacingPips = 100 |
| 最大加仓次数 | 防止过度加仓 | MaxAddOrders = 5 |
| 同向确认 | 确保趋势延续 | 可选：M3均线多头排列 |

## 三、仓位管理（反马丁核心）

### 3.1 加仓手数计算

```
公式：NextLot = PrevLot × AddLotRatio

参数：
- AddLotRatio = 0.8（递减系数，必须<1）
- 首仓手数 = 1.0手（示例）
- 第1次加仓 = 1.0 × 0.8 = 0.8手
- 第2次加仓 = 0.8 × 0.8 = 0.64手
- 第3次加仓 = 0.64 × 0.8 = 0.51手
- 以此类推...

总仓位上限：MaxTotalLots = 3.5手（首仓+5次加仓总和约3.2手）
```

### 3.2 与马丁的对比

| 次数 | 马丁（倍投） | 反马丁（递减） |
|------|-------------|---------------|
| 首仓 | 1.0 | 1.0 |
| 第1加 | 2.0 | 0.8 |
| 第2加 | 4.0 | 0.64 |
| 第3加 | 8.0 | 0.51 |
| 第4加 | 16.0 | 0.41 |
| 第5加 | 32.0 | 0.33 |
| **总仓位** | **63手（爆仓）** | **3.69手（可控）** |

## 四、止损机制（三层防护）

### 4.1 第一层：首仓硬止损

```
首仓止损 = 开仓价 ± StopLossPips

参数：
- StopLossPips = 80（XAU/USD约800美元/手）
- 作用：保护首仓，防止大亏损
- 特性：固定，不移动
```

### 4.2 第二层：移动保本止损

```
触发条件：当 newest加仓单盈利达到 BreakevenTriggerPips

操作：
1. 将所有订单（首仓+所有加仓）的止损移至整体成本价
2. 此后无论如何不会亏损

参数：
- BreakevenTriggerPips = 50
- 作用：锁定本金，确保不亏
```

### 4.3 第三层：移动止盈止损（Trailing）

```
触发条件：价格继续向有利方向移动

操作：
1. 当价格从最高点回撤 TrailingStartPips 时
2. 启动追踪止损，跟随价格移动
3. 所有订单同步移动止损

参数：
- TrailingStartPips = 100
- TrailingStepPips = 50
- 作用：让利润奔跑，同时保护已得利润
```

### 4.4 第四层：账户总风险控制

```
硬止损线：当账户浮亏达到 AccountRiskPercent% 时，全部平仓

参数：
- AccountRiskPercent = 3%（建议不超过5%）
- 作用：极端行情保护，防止黑天鹅
- 优先级：最高，无视其他条件
```

## 五、止盈机制

### 5.1 分批止盈（金字塔出场）

```
方案A：固定目标分批止盈
- 第1目标：整体盈利达 RRR 1:1，平掉20%仓位
- 第2目标：整体盈利达 RRR 1:2，平掉30%仓位
- 第3目标：整体盈利达 RRR 1:3，平掉剩余50%

方案B：移动止盈（推荐）
- 不主动止盈，只用Trailing Stop
- 让趋势自然决定出场点
- 适合趋势行情，盈亏比可达5:1以上
```

### 5.2 时间止盈

```
条件：订单持仓超过 MaxHoldHours 小时

操作：
- 如果盈利 > MinProfitPips，平仓
- 如果盈利 < MinProfitPips，持有等待

参数：
- MaxHoldHours = 48
- MinProfitPips = 30
- 作用：防止订单长期占用资金
```

## 六、应对不同行情的机制

### 6.1 趋势行情（最佳）

```
特征：价格沿一个方向持续移动

EA表现：
- 多次触发加仓，持仓量逐渐增加
- 移动止损不断上移/下移
- 利润持续放大

参数优化：
- AddSpacingPips = 150（放宽间距，减少加仓频率）
- MaxAddOrders = 8（增加加仓次数，抓大趋势）
```

### 6.2 震荡行情

```
特征：价格在区间内来回波动

EA表现：
- 可能触发1-2次加仓后回撤
- 触及移动保本止损，全部平仓
- 小盈利或小亏损出场

参数优化：
- AddSpacingPips = 80（收紧间距，快速加仓）
- MaxAddOrders = 3（限制加仓次数）
- 配合：首仓选择震荡策略（你手动判断）
```

### 6.3 单边极速行情（V型反转）

```
特征：价格瞬间大幅反向

EA表现：
- 可能来不及加仓或刚刚加仓就反转
- 首仓止损被触发，全部平仓
- 亏损可控（首仓止损范围内）

保护机制：
- 账户总风险3%硬止损
- 避免连续亏损加仓（反马丁天然优势）
```

### 6.4 隔夜/周末风险

```
机制：
- 周五收盘前1小时，自动平仓所有订单
- 或：将止损收紧至保本位置
- 重大数据公布前30分钟，暂停新开加仓

参数：
- CloseBeforeWeekend = true
- CloseBeforeHighImpactNews = true
```

## 七、补仓机制（特殊场景）

### 7.1 什么情况下补仓？

反马丁原则上**不补仓**，但如果首仓被扫止损后，你想重新进场：

```
机制：首仓止损后，EA进入观察模式

条件：
1. 价格回到原开仓价附近（±20点内）
2. M3周期出现新的入场信号（如均线金叉/死叉）
3. 你手动确认重新开仓

操作：
- EA将新订单视为新的首仓
- 重新启动加仓流程

限制：
- 同一方向24小时内最多重开2次
- 防止反复止损消耗本金
```

### 7.2 加仓单被止损后的处理

```
场景：加仓后价格回撤，触及移动保本止损

操作：
1. 所有订单（包括首仓）全部平仓
2. 记录本次交易结果到日志
3. EA进入待机模式，等待新的首仓

不操作：
- 不会立即反向开仓
- 不会追加补仓
```
